You are running the **quick brief** — a single-file alternative to the full 5-file `client-brief/` interview. Use this for small one-off jobs (a homepage tweak, a single product page rebuild, a metafield migration) where the full brief is overkill.

### Rules

- One question at a time. Wait for each answer.
- Write to `client-brief-QUICK.md`, overwriting it with the captured answers.
- If `client-brief/` has substantive content already, ask whether the user wants to use that instead before continuing.

### The 8 questions

1. **What are we building or fixing in this engagement?** (one sentence)
2. **Who's the ideal buyer?** (one sentence — the level of specificity matters more than length)
3. **What's the #1 outcome that matters?** (purchase, subscription, email, page views, etc.)
4. **Brand voice in 3 adjectives.** Plus one phrase you'd never use.
5. **Visual direction — pick one or describe:** clean/minimal, bold/editorial, earthy/artisan, luxury/refined, playful/vibrant. Reference store URLs welcome.
6. **What's the hero message?** The single most important thing the homepage (or target page) needs to communicate.
7. **Any constraints?** Budget, timeline, apps that must stay, anything we shouldn't touch.
8. **What does "done" look like for this job?** A specific deliverable.

After all 8 answers are in, write `client-brief-QUICK.md` and recommend the next command (likely `/homepage-from-brief` or whichever build command fits the job).

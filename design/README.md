# design/

Home for all visual design references for this client. This folder is separate from `client-brief/` (strategy and audience) and `store-data/` (theme files). Its job is holding the visual inputs that drive implementation.

## What goes here

| Folder / File | What it holds |
|---|---|
| `mockups/` | PNGs, screenshots, Figma exports. Drop images here before running `/design-intake` or `/implement-mockup-module`. |
| `assets/` | SVG wordmarks, logos, icons, any vector art the client provides. |
| `brand-system.md` | Palette hex codes, fonts, typography rules, voice rules, visual references and anti-references. Generated by `/brand-system` or filled manually. |
| `page-specs/` | Module-by-module implementation specs (e.g. `homepage.md`). Generated by `/design-intake`. Referenced by `/implement-mockup-module` and `/homepage-from-brief`. |

## The pipeline

```
Client provides mockup
        ↓
design/mockups/homepage.png
        ↓
/design-intake   →   design/page-specs/homepage.md
                              ↓
/implement-mockup-module hero   →   theme implementation
```

If no mockup exists yet, run `/brand-system` to capture the visual identity first. That gives Claude enough to propose a direction via `/homepage-from-brief`, even without a pixel-perfect mockup.

## What this is NOT

- Not for theme files — those live in `store-data/theme/` (gitignored)
- Not for client strategy — that lives in `client-brief/`
- Not for session notes — those live in `docs/`
